预取器控污限流防线

1. 已驻留过滤 (Resident)

发出预取前，若目标下一行已经存储在 Cache 中或已经在 Miss 队列（MSHR）里等待填充，立即丢弃，防范总线重复申请。

2. 组污染控制 (Throttle)

若目标 Set 组里存在已被预取但未被 CPU 命中读取的行，判定程序当前非顺序流，强行关闭当前组的预取，防止有效数据被逐出。

3. 推测避让让路 (Cancel)

当 Set 满且发生急迫的常规 Miss 时，Cache 启动自救，强行注销（Cancel）在该 Set 下在途但未完成的推测预取，为真实需求让位。

预取质量指标追踪 (Prefetch Stats Counter)

指标类型 计数器名称 当前计数值 物理作用解释
预取发起 prefetch_requests 0 发生了 Demand Miss，触发预取决策的总次数。
预取发出 prefetch_issued 0 成功通过所有限流门控，真正向内存发出加载的次数。
有用命中 prefetch_useful_hits 0 预取进来的 Cache Line，在被驱逐前成功被正常 Load 命中的次数（性能正收益源泉）。
无用驱逐 prefetch_unused_evictions 0 预取进来的 Cache Line，一次都没被读写就被冷落淘汰，白白占用位置（污染指标）。
驻留丢弃 prefetch_dropped_already_resident 0 检查到下一行已经在 Cache 或 MSHR 中而主动放弃预取。
限流丢弃 prefetch_dropped_set_throttle 0 由于 Set 组污染超标（未用预取数>=1）或 MSHR 资源满而强行抛弃的次数（防卫机制指标）。
