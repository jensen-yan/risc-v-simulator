LSU 三大宏观设计支柱

1. 写操作隔离 (Buffer)

Store 指令计算出的数据决不能直接写进内存或 Cache，必须先寄存在 Store Buffer 中，直到该 Store 顺序提交时才刷入内存。

2. 读操作抢跑 (Speculate)

Load 指令的内存访问延迟极高。为了隐藏延迟，LSU 允许 Load 指令在地址不冲突的前提下，越过前方的 Store 提前读取 Cache。

3. 违规自学习 (Learn)

盲目抢跑若撞上“物理地址重叠”，会导致读脏数据。LSU 在回滚的同时会用黑名单对记录下它们，使 Load 下次学会挂起等待。
